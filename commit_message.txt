refactor(config, updates): Standardize JSON schema and update handling

This commit introduces several improvements to standardize the application configuration schema and streamline update handling logic.

- **Standardized JSON Schema (`config/schema.json`):**
  - Introduced `common_properties` definition to ensure consistent validation rules across all application types (e.g., `name`, `checksum_url`, `gpg_key_id`).
  - Applied `common_properties` to `appimage`, `custom`, `direct_download`, `github_release`, and `script` definitions.
  - Reformatted the entire schema for better readability and maintainability.

- **Standardized "Up to Date" Handling (`src/core/updates.sh`):**
  - Created `updates::handle_up_to_date` function to centralize the display of "up to date" messages and increment the `up_to_date` counter.
  - Replaced direct calls to `interfaces::print_ui_line` and `counters::inc_up_to_date` with `updates::handle_up_to_date` in `check_script`, `check_github_release`, `check_direct_download`, `check_appimage`, and `handle_custom_check`.
  - Refactor only; no user-visible change to messaging or counters.

- **Global Version Filename Regex:**
  - Defined `VERSION_FILENAME_REGEX` in `src/core/globals.sh` for consistent version extraction from filenames.
  - Updated `packages::extract_deb_version`, `updates::check_direct_download`, `updates::check_appimage`, and `versions::extract_from_regex` to utilize this new global regex.
  - Intent: unify prior adâ€‘hoc extraction patterns with a single canonical matcher to reduce drift and false negatives.
  - Matches examples: `v1.2.3`, `1.2.3-beta`, `myapp-1.2.3.tar.gz` (captures `1.2.3` / `1.2.3-beta`). Excludes non-semver or too-short forms like `1.2` (missing patch) or build tags like `build123` lacking dotted core.
  - Supports: optional leading `v`, pre-release suffixes (e.g., `-alpha`, `-beta.1`, `-rc.0`), and build metadata after `+` (ignored downstream by normalization which strips `+...`).
  - Compatibility: replaces previous scattered filename regex usages with no intended behavior change for existing configs; normalization still trims leading `v` and coerces prerelease to dpkg style.
  - Known exceptions: none identified requiring a divergent pattern at this time.
  - Verification: exercised against existing filenames in repository and related tests; no regressions observed.

- **Configuration File Reformatting:**
  - Applied consistent JSON formatting to `config/conf.d/cursor.json`, `config/conf.d/fastfetch.json`, and `config/conf.d/ollama.json`.
  - Formatting only; no semantic or behavioral changes to these JSON files.
